Fangs Builder

Github Actions

My first foray into the world of front-end dev. This is a project to make it possible to people to share their favorite build configurations for an indie game called Fangs - think of website like mobafire, op.gg, dotabuilds, icyveins, etc. People could compose a build on the website, then just click a button and get a shareable link that would show others the build. The UI was made to be as similar as possible to the in-game functionality at the time.
I chose Solid Js because it seemed like a great framework without too much bloat and primegean made a video on it as well. I also was not planning on having a backend so it just made sense.
Hosted in Azure Storage using the Static Website feature because it's very cheap, with deployment being done automatically using github Actions as they offer free build agents for open source projects.

An iteration on the previous Fangs Builder project. This time with a backend.
Supports features such as: OAuth2 Discord(checks role and issue JWT), Persisting Builds, Voting on Builds, Admin page for data entry/edit, Map Browser/Drawing Tool.
Front End made using Angular and Typescript and hosted in Azure Storage Static Websites, served through Azure CDN, UI infrastructure provisioned using terraform and deployment done with Github Actions.
Backend is a C# ASP.NET Core Web Api hosted on a ubuntu VM in azure, configuration and deployment done by manually sshing into the machine for experimentation: installed nginx and proxied to the manually deployed api.
